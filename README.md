# Описание проекта
В рамках практической работы было разработано WPF-приложение SalaryCalculator, предназначенное для расчёта заработной платы преподавателя с учётом почасовой ставки и подоходного налога.

# Основные возможности приложения:

## Выбор должности:

Ассистент – 150 руб/час

Доцент – 250 руб/час

Профессор – 350 руб/час

## Ввод количества отработанных часов.

## Выбор ставки подоходного налога: 13% или 0%.

## Автоматический расчёт:

## Начисленной суммы (часы × ставка)

## Суммы налога (начисленная сумма × ставка налога / 100)

## Отображение результата в соответствующих полях.

## Обработка ошибок ввода:

Некорректный формат часов

Отрицательное количество часов

Отсутствие выбора должности или налоговой ставки

# Структура проекта:

SalaryCalculator.WPF – главное WPF-приложение (интерфейс и бизнес-логика).

SalaryCalculator.Tests – проект модульных тестов на MSTest, проверяющий корректность работы методов класса SalaryCalculator.

# Пример работы программы
После запуска приложения пользователь выбирает должность, вводит количество часов, выбирает налоговую ставку и нажимает кнопку «Вычислить».

# Результаты тестирования
Для проверки корректности работы методов класса SalaryCalculator были разработаны Unit-тесты на фреймворке MSTest.

Перечень тестов:

Calculate_Assistant_10Hours_13PercentTax_ReturnsCorrectGrossAndTax – проверка расчёта для ассистента (10 ч, налог 13%).

Calculate_Professor_15Hours_0PercentTax_ReturnsGrossAndNoTax – проверка расчёта для профессора (15 ч, налог 0%).

Calculate_ZeroHours_ReturnsZeroGrossAndTax – проверка граничного случая (0 часов).

Calculate_NegativeHours_ThrowsArgumentExceptionWithCorrectMessage – проверка выброса исключения при отрицательных часах.

Calculate_InvalidTaxRateGreaterThan100_ThrowsArgumentExceptionWithCorrectMessage – проверка выброса исключения при недопустимой налоговой ставке.

Calculate_FractionalHours_CalculatesCorrectly – проверка корректности расчёта при дробном количестве часов.
